_Quali delle seguenti affermazioni su RC4 e AES sono vere?2
*RC4 non richiede padding perché il keystream ha la stessa lunghezza del messaggio da cifrare.
*AES, con l’opzione -nopad, può cifrare un messaggio solo se la sua lunghezza è un multiplo della lunghezza del blocco.
-RC4 richiede padding per cifrare correttamente messaggi più corti del keystream.
-AES è un algoritmo di cifratura di tipo stream, mentre RC4 è a blocchi.
-Sia RC4 che AES richiedono che il messaggio sia sempre un multiplo della lunghezza del blocco per essere cifrato.

_Quali tecniche migliorano la sicurezza complessiva di IPsec e TLS?3
*Abilitare PFS (Perfect Forward Secrecy) per le sessioni.
*Aggiornare regolarmente i certificati TLS con chiavi di lunghezza adeguata.
-Disabilitare il supporto a TLS 1.3.
-Usare IPsec solo in transport mode.
-Usare protocolli di hash più veloci per ridurre la possibilità di attacchi brute-force.
*Disabilitare versioni obsolete come TLS 1.0 e IPsec con DES.
-Usare sempre e solo AH in IPsec.
-Cambiare spesso le SPD per evitare attacchi di tipo brute-force.

_Indica quali delle seguenti affermazioni sull'uso di TLS e STARTTLS nella posta elettronica sono vere:1
*Nei protocolli IMAP e POP3, l'uso di STARTTLS implica che la connessione viene prima stabilita in chiaro e solo successivamente, su richiesta del client, parte la negoziazione di un canale sicuro.
- Il comando "EHLO" in SMTP viene utilizzato dal client per avviare una connessione TLS sicura.

__Alice e Bob vogliono configurare un canale di comunicazione sicuro utilizzando IPSec. A questo proposito Bob esegue i seguenti comandi:
ip xfrm state flush 
ip xfrm policy flush 
ip xfrm state add sic IP-Alice dst IP-Bob proto esp spi 0x1000 enc aes Oxaa223344556677889900aabbccddeeff 
ip xfrm state add sic IP-Bob dst IP-Alice proto esp spi 0x2000 enc aes 0xbb223344556677889900aabbccddeeff 
ip xfrm policy add src IP-Alice dst IP-Bob dir in tmpl proto esp mode transport level required ip xfrm policy add src IP-Bob dst IP-Alice dir out tmpl proto esp mode transport level required
Seleziona le risposte corrette:_3
*Vengono create 2 SA e 2 SP
*Le SA contengono 2 SPI differenti perchè identificano la direzione
*Le Security Policies (SP) configurate stabiliscono che il traffico tra Alice e Bob deve essere cifrato utilizzando ESP in modalità transport, senza coinvolgere l’intero pacchetto IP.
-Le SA configurate utilizzano lo stesso SPI per entrambe le direzioni, perché l’algoritmo AES richiede univocità solo a livello di chiave crittografica.
-Le Security Associations (SA) configurate permettono di proteggere il traffico solo nella direzione da Bob a Alice, mentre il traffico nella direzione opposta rimane non protetto.

__Alice e Bob vogliono configurare un canale di comunicazione sicuro, ma utilizzano configurazioni differenti. Alice sceglie ESP con AES-128-GCM (AEAD) , mentre Bob utilizza AH con HMAC-SHA1.
Alice quindi esegue i seguenti comandi:
ip xfrm state add sic IP-Alice dst IP-Bob proto esp spi 0x1000 aead "rfc4106 (gcm (aes)) "0x223344556677889900aabbccddeeffaabbccdd 128
ip xfrm state add sc IP-Bob dst IP-Alice proto esp spi 0x2000 aead "rfc4106 (gcm (aes)) "0xbb223344556677889900aabbccddeeffaabbccdd 128
ip xfrm policy add src IP-Alice dst IP-Bob dir out tmpl proto esp mode transport level required
ip xfrm policy add sic IP-Bob dst IP-Alice dir in tmpl proto esp mode transport level required
Mentre Bob esegue i seguenti comandi:
ip xfrm state add sc IP-Alice dst IP-Bob proto ah spi 0x1000 auth "hmac (shal) " 0xaa1222333444555666777888999000aaabbbcccd
ip xfrm state add src IP-Bob dst IP-Alice proto ah spi 0x2000 auth "hmac (shal)" 0xbb1222333444555666777888999000aaabbbcccd
ip xfrm policy add src IP-Bob dst IP-Alice dir out tmpl proto ah mode transport level required
ip xfrm policy add src IP-Alice dst IP-Bob dir in tmpl proto ah mode transport level required
Seleziona le affermazioni corrette:_3
*Sono presenti in totale 4 SA e 4 SP
-Le SP sono 2 in totale perché, a differenza delle SA, sono bidirezionali
-Bob ha sbagliato la sua configurazione perchè non ha specificato l’algoritmo crittografico da utilizzare
*La configurazione è vantaggiosa perchè protegge sia l’integrità del messaggio che la segretezza del payload usando un singolo algoritmo e una singola chiave
