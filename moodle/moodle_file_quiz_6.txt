// question: 1  name: Bob gestisce un server che deve essere accessibile sia per connessioni VPN sicure che per accesso HTTPS per i clienti. Quali differenze fondamentali tra IPsec e TLS dovrebbero influenzare la sua configurazione?
// [tag:ipsec] [tag: tls] [tag: differenze] [tag: sicurezza] 
::Bob gestisce un server che deve essere accessibile sia per connessioni VPN sicure che per accesso HTTPS per i clienti. Quali differenze fondamentali tra IPsec e TLS dovrebbero influenzare la sua configurazione?::[html]<p><strong>Bob gestisce un server che deve essere accessibile sia per connessioni VPN sicure che per accesso HTTPS per i clienti. Quali differenze fondamentali tra IPsec e TLS dovrebbero influenzare la sua configurazione?</strong></p>{
	~%33.33333%[moodle]IPsec opera a livello di rete, mentre TLS opera a livello di trasporto, rendendo TLS più adatto per la protezione di applicazioni specifiche all’interno dello stesso server.
	~%33.33333%[moodle]IPsec può fornire sicurezza per tutto il traffico di rete (unicast), inclusi protocolli non orientati alla connessione, mentre TLS protegge solo protocolli basati su TCP.
	~%33.33333%[moodle]L'autenticazione in TLS è spesso basata su certificati, mentre IPsec può supportare autenticazione con chiavi precondivise o certificati.
	~[moodle]TLS può essere utilizzato per proteggere il traffico multicast, mentre IPsec no.
	~[moodle]IPsec è ottimale per la protezione di pagine web dinamiche come HTTPS, mentre TLS non è progettato per questo.
}

// question: 6  name: Quali tra le seguenti affermazioni riguardanti il principio di Kerckhoffs sono vere?
// [tag:crypto] [tag: algoritmi] 
::Quali tra le seguenti affermazioni riguardanti il principio di Kerckhoffs sono vere?::[html]<p><strong>Quali tra le seguenti affermazioni riguardanti il principio di Kerckhoffs sono vere?</strong></p>{
	~%50.00000%[moodle]Il principio di Kerckhoffs afferma che un sistema crittografico deve essere sicuro anche se l’algoritmo è pubblico, purché la chiave sia segreta.
	~%50.00000%[moodle]L’applicazione del principio di Kerckhoffs ha portato alla trasparenza nello sviluppo degli algoritmi crittografici moderni come AES.
	~[moodle]La segretezza dell’algoritmo garantisce sempre un livello aggiuntivo di sicurezza rispetto alla trasparenza dell’algoritmo.
	~[moodle]Il principio di Kerckhoffs richiede che l’algoritmo di crittografia e le chiavi siano mantenuti segreti per garantire sicurezza.
	~[moodle]Le vulnerabilità negli algoritmi pubblici sono inevitabili e non possono essere mitigate con il solo uso di chiavi sicure.
}

// question: 7  name: Indica quali delle seguenti affermazioni su firewall, proxy e DMZ sono corrette?
// [tag:firewall] [tag: proxy] [tag: DMZ] 
::Indica quali delle seguenti affermazioni su firewall, proxy e DMZ sono corrette?::[html]<p><strong>Indica quali delle seguenti affermazioni su firewall, proxy e DMZ sono corrette?</strong></p>{
	~%25.00000%[moodle]Un forward proxy si trova tra i client di una rete esterna e i server esterni, inoltrando le richieste dei client.  
	~[moodle]Un reverse proxy si limita a bilanciare il carico tra più server interni, senza fornire alcuna funzionalità di sicurezza aggiuntiva.  
	~%25.00000%[moodle]L'utilizzo di una DMZ (De-Militarized Zone) permette di isolare i server pubblici dalla rete interna, aumentando la sicurezza.
	~[moodle]In un'architettura firewall con DMZ, il gateway è sempre posizionato all'interno della rete interna per garantire una maggiore protezione.
	~%25.00000%[moodle]Un firewall "a tre gambe" è un'architettura che usa un unico gateway per generare una DMZ e isolare la rete interna.  
	~[moodle]Il mascheramento degli indirizzi IP interni (NAPT) è una funzionalità esclusiva dei proxy application layer.
	~%25.00000%[moodle]I firewall possono essere implementati come software, appliance hardware o servizi cloud.  
	~[moodle]Un firewall che contiene componenti di tipo application layer filter è in grado di ispezionare il traffico crittografato senza necessità di decifrarlo.
	~[moodle]I firewall possono essere implementati come app mobile o servizi cloud.
	####[moodle]Ripetere concetti su firewall, proxy e DMZ [Slide 09]
}
