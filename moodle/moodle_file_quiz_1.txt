// question: 3  name: Quali tra le seguenti affermazioni sul padding nella crittografia a blocchi sono vere?
// [tag:crypto] [tag: simmetrica] 
::Quali tra le seguenti affermazioni sul padding nella crittografia a blocchi sono vere?::[html]<p><strong>Quali tra le seguenti affermazioni sul padding nella crittografia a blocchi sono vere?</strong></p>{
	~%50.00000%[moodle]Il padding garantisce che l’input della cifratura a blocchi sia un multiplo della lunghezza del blocco.
	~%50.00000%[moodle]In CBC, il padding viene aggiunto solo al blocco finale, indipendentemente dalla lunghezza del messaggio.
	~[moodle]Il padding è necessario in tutte le modalità operative della crittografia a blocchi.
	~[moodle]Il padding è sempre utilizzato per garantire integrità dei dati cifrati.
	~[moodle]Algoritmi come AES in modalità CBC non richiedono padding se la lunghezza del messaggio è nota.
}

// question: 4  name: Quali tra le seguenti affermazioni sul protocollo ESP di IPsec sono vere?
// [tag:esp] [tag: ipsec] 
::Quali tra le seguenti affermazioni sul protocollo ESP di IPsec sono vere?::[html]<p><strong>Quali tra le seguenti affermazioni sul protocollo ESP di IPsec sono vere?</strong></p>{
	~%50.00000%[moodle]ESPv1+ fornisce riservatezza cifrando il payload del pacchetto IP.
	~%50.00000%[moodle]ESPv2+ può essere configurato per garantire autenticità e integrità dei dati tramite algoritmi di keyed-hash.
	~[moodle]ESPv1+ crittografa sempre l’intero pacchetto IP, inclusi gli header, indipendentemente dalla modalità utilizzata.
	~[moodle]ESPv2 è progettato per funzionare esclusivamente in modalità Tunnel.
	~[moodle]ESPv3+ non può essere utilizzato insieme al protocollo AH.
}

// question: 5  name: indica quali delle seguenti affermazioni sul laboratorio relativo ai firewall sono corrette:

	~%25.00000%[moodle] Netfilter/IPtables è un software che fornisce funzionalità di manipolazione e filtraggio di pacchetti IP all'interno del kernel Linux.
	~[moodle]La catena INPUT di IPtables si applica esclusivamente ai pacchetti in uscita dalla macchina su cui è installato IPtables, mentre la catena OUTPUT gestisce i pacchetti in ingresso.

	~%25.00000%[moodle]La catena OUTPUT di IPtables si applica esclusivamente ai pacchetti in uscita dalla macchina su cui è installato IPtables, mentre la catena INPUT gestisce i pacchetti in ingresso.

// question: 7  name: Indica quali affermazioni sui diversi tipi di firewall e il loro funzionamento sono corrette?
// [tag:firewall] 
::Indica quali affermazioni sui diversi tipi di firewall e il loro funzionamento sono corrette?::[html]<p><strong>Indica quali affermazioni sui diversi tipi di firewall e il loro funzionamento sono corrette?</strong></p>{
	~%25.00000%[moodle]I firewall packet filter operano a livello di rete (L3) e ispezionano l'header di ogni pacchetto IP.  
	~[moodle]I circuit-level gateway esaminano il contenuto dei pacchetti a livello applicativo per identificare e bloccare minacce specifiche.  
	~%25.00000%[moodle]Un proxy application-aware capisce il protocollo applicativo e può quindi applicare regole di sicurezza più specifiche.

	~[moodle]I componenti di firewall che operano a livello di trasporto (L4) possono autenticare gli utenti direttamente basandosi sulle informazioni contenute nei pacchetti TCP/UDP senza richiedere modifiche alle applicazioni.  
	~%25.00000%[moodle]I componenti di firewall che operano usando il protocollo SOCKS creano un "circuito" a livello di trasporto tra client e server, senza necessariamente ispezionare il contenuto dei dati applicativi.
